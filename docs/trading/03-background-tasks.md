# 주문 이후 백그라운드 태스크

## 실시간/준실시간 태스크
- Mock Exchange 이벤트 소비: 접수/거부/체결/취소 이벤트 처리
- 잔고 반영: 가용수량/주문중수량/예수금 업데이트
- 리스크 체크: 주문 한도, 계좌 위험도 재평가
- 알림 발송: 앱 푸시/웹소켓
- 감사로그 기록: 누가/언제/무엇을 변경했는지 추적

## 비동기 배치 태스크
- 일별 거래내역 집계
- 수수료/세금 정산 보정
- 대사(reconciliation): 내부 원장 vs Mock Exchange 체결 데이터 비교
- 이상거래 탐지/리포트 생성

## 장애 대비 태스크
- 재처리 큐(DLQ) 복구
- 내부 이벤트 처리 실패 시 재시도(backoff)
- 이벤트 중복/역순 수신 시 정합성 보정
